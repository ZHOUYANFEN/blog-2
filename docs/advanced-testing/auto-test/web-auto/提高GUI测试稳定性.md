# 提高 GUI 测试稳定性

**GUI 不稳定因素**：

+ 非预计的弹出对话框
+ 页面控件属性的细微变化
+ 被测系统的A/B测试
+ 随机的页面延迟造成控件识别失败
+ 测试数据问题

## 不稳定解决思路

### 非预计的弹出对话框

**包含场景**：

+ GUI 自动化测试用例执行过程中，操作系统弹出的非预计对话框
+ 被测软件本身也有可能在非预期的时间弹出预期的对话框

**解决方案**：

+ 当自动化脚本发现控件无法正常定位，或者无法操作时，GUI 自动化框架自动进入“异常场景恢复模式”
+ 在“异常场景恢复模式”下，GUI 自动化框架依次检查各种可能出现的对话框，一旦确认了对话框的类型，立即执行预定义的操作（比如，单击“确定”按钮，关闭这个对话框），接着重试刚才失败的步骤
